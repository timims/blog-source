## 背景

过去我在学习和工作中积累了不少和数据库、后端、系统环境有关的经验，但很多内容都分散在命令行历史、笔记软件和临时聊天里，复用效率并不高。

我希望有一个稳定、低成本、可长期维护的地方，把这些工程实践沉淀下来。相比临时笔记，博客更适合形成完整文章，也更适合长期检索和回顾。

## 问题

技术积累如果没有结构化输出，通常会遇到几个问题：

1. 同一个问题过一段时间还要重新查一遍。
2. 一次解决的经验没有被抽象成通用方法。
3. 遇到复杂问题时，很难回溯当时的背景、原因和取舍。

对我来说，真正有价值的内容不是零散结论，而是完整的问题处理过程。

## 解决方案

这个博客会坚持工程实践导向，重点记录下面几个方向：

### 数据库

- TimescaleDB 的使用经验
- PostgreSQL 查询、索引与性能分析
- 实际业务场景中的时序数据建模

### 后端

- Java 服务开发中的设计与优化
- 接口治理、并发处理与性能问题定位
- 常见工程问题的解决思路

### 系统

- Arch Linux 环境配置
- 开发工具链问题排查
- 本地开发、部署和脚本自动化经验

### 工程实践

- 项目部署流程
- 架构演进中的取舍
- 真实踩坑记录与复盘总结

为了保证内容质量，我后续的文章会尽量遵循统一结构：

1. 背景
2. 问题
3. 解决方案
4. 结果
5. 总结

## 结果

目前博客已经完成了基础搭建，包含以下能力：

- 基于 Hexo 和 Butterfly 主题
- 支持分类、标签、归档和本地搜索
- 使用 GitHub Pages 托管
- 使用 GitHub Actions 自动部署

这意味着后续我只需要专注于写文章，而不需要把精力放在额外的服务维护上。

## 总结

这篇文章算是这个博客的正式开始。

后续我会优先发布这些主题的内容：

1. TimescaleDB 与 PostgreSQL 的实践记录
2. Java 后端开发中的性能与工程问题
3. Arch Linux 和开发环境相关的踩坑经验

如果一篇文章能让未来的我少排查一次问题，那它就已经有价值了。
